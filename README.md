# AI-Agent-Authentication-and-Authorization

IETF125 side meeting for the topic of AI Agent Authentication can Authorization Agent security is frequently mentioned, yet clear requirements have not been fully defined.

The purpose of this side meeting is to clarify the needs and challenges related to authentication and authorization in agent networks, thereby accelerating the advancement of agent security.

Meanwhile, we have also taken note of other side meetings and discussion topics related to agent security including BoF Catalist and AIprotocol & AI angent security.

Analysis of the core issues, the discussions can be categorized into the following four key problem areas:

Lack of Identity and Trust: There is an absence of unified identity standards, verification methods, and cross-domain trust foundations for AI agents. An agent's "credentials" cannot be mutually recognized across different platforms.

Inflexible Authorization Mechanisms: Traditional authorization frameworks (e.g., OAuth) face challenges when applied to AI agents, including misinterpretation of intent, insufficient permission granularity, and a disconnect between authorization and specific actions. This leads to risks such as over-privileged access or hallucination injection.

Loss of Security Context: In complex, cross-domain, multi-agent invocation chains, critical security context information—such as user identity, agent identity, and operational intent—is difficult to transmit and maintain consistently.

Difficulties in Discovery and Interoperability: The lack of standardized protocols for agent discovery, metadata description, and session management hinders effective collaboration across organizations and ecosystems.
